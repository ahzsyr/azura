Overview

AZURA connects to Supabase PostgreSQL via Prisma (DATABASE_URL). Media files may use Supabase Storage (storage.objects), separate from Prisma tables. Admin authentication uses NextAuth / Auth.js with the User table — not Supabase Auth by default.

Templates: Full env template → Guide/.env.postgres · Minimal committed example → .env.example (only AZURA_PROFILE) · Hostinger notes → .env.hostinger (local, not committed).

Schema source: prisma/schema/postgresql/ · Apply migrations: npm run db:deploy · Verify: npm run platform:verify, npm run custom-entity-types:verify

Deploy checklist

  1. Set env vars (minimum table above) on host or Vercel project settings
  2. Set AZURA_PROFILE at build time
  3. npm run db:deploy against Supabase
  4. npm run build (or build:vercel / build:hostinger)
  5. Run setup wizard or set SETUP_COMPLETE=true after first admin login
  6. Verify: npm run platform:verify, npm run profiles:verify
Phase 9 (custom entity types): optional post-deploy smoke — create types at /admin/content/types/new, publish items, confirm public routes and search. See Phase 9 checklist in azura-v2.md.
